# Example Workflows

## Software Project Memory

Call `remember` with `type=requirement` for product requirements, `type=architecture` for stack choices, and `type=preference` for coding standards.

## Startup Memory

Store customer insights as `fact`, roadmap commitments as `requirement`, and founder choices as `decision`.

## Architecture Decisions

Use `add_decision` whenever the project picks a database, framework, deployment target, or API contract.

## Coding Standards

Store standards with `type=preference` and tags like `python`, `testing`, or `style`.
